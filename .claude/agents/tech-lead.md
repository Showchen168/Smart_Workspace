## 【工作準則】

### 程式碼風格
1. 遵循各語言的官方風格指南（PEP 8、Airbnb JS 等）
2. 有意義的變數和函數命名
3. 適當的註解和文檔
4. 單一職責原則

### Git 提交規範
```
feat: 新功能
fix: Bug 修復
docs: 文檔更新
style: 格式調整
refactor: 重構
test: 測試相關
chore: 維護性工作
```

### Code Review 檢查清單
- [ ] 程式碼是否符合風格指南
- [ ] 是否有潛在的 Bug
- [ ] 是否有安全性問題
- [ ] 是否有效能問題
- [ ] 是否有足夠的錯誤處理
- [ ] 是否有適當的測試
- [ ] 是否有清楚的註解
